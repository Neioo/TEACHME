# TEACHME.md

TEACHME.md is an engineering thinking framework designed to teach **how to think like a software engineer**, not just how to write code.

It focuses on reasoning, decision-making, system design, and deep code understanding.

This is not a tutorial.
This is not a framework guide.
This is a structured way of thinking about software.

---

## Why this exists

Most learning resources focus on:

* Syntax
* Framework usage
* Step-by-step implementation

This creates "tool users", not engineers.

TEACHME.md is built to address a different gap:

> The ability to reason about why something should be built, not just how to build it.

---

## Core Idea

Engineering is not coding.

Engineering is decision-making under constraints.

Every solution is a tradeoff.

---

## Engineering Thinking Framework (Always On)

All explanations follow this structure:

### 1. Problem

What problem are we solving and why does it matter?

### 2. Constraints

What limitations exist? (technical, business, time, scale, cost)

### 3. Options

What are the possible approaches?

### 4. Tradeoffs

What are the benefits, costs, and risks of each option?

### 5. Decision

Which option is chosen and why?

### 6. Implementation

Only after reasoning is complete is implementation discussed.

---

## Code Dissection Framework (Adaptive Depth)

Code is not treated as “magic syntax”.

Every piece of code is broken down based on depth level.

### Level 0 — Intent

What does this do at a high level?

### Level 1 — Component Breakdown

Break major parts into understandable pieces.

### Level 2 — Full Dissection

Explain:

* Every line
* Every keyword
* Every function call
* Ownership (language vs framework vs application)
* Runtime behavior
* What can and cannot be changed

### Level 3 — Deep Engineering Analysis

Explain:

* Why this design exists
* Alternatives
* Tradeoffs
* Hidden assumptions
* Architectural consequences

---

## First Encounter Rule

When a concept appears for the first time, it is automatically explained in **at least Level 2 depth**.

This includes:

* Framework features
* Design patterns
* New syntax structures
* Architectural concepts

---

## Teaching Philosophy

This framework assumes:

* Understanding matters more than speed
* Reasoning matters more than memorization
* Tradeoffs matter more than answers
* Code is a result of decisions, not the starting point

---

## Target Audience

This is designed for:

* Developers learning system design
* Engineers improving architectural thinking
* Builders working on real-world scalable systems
* Anyone trying to move beyond “tutorial-level coding”

---

## Core Principle

> If you can explain why a system exists, you can rebuild it anywhere.

---

## Status

This is an evolving framework.
It will be refined as thinking improves.

## Usage & Derivatives

TEACHME.md is designed to be used, modified, and adapted across different projects, teams, and learning environments.

It is intentionally not tied to any specific language, framework, or architecture.

---

## How to Use This Framework

You can:

* Apply it to your own projects as a thinking guide
* Modify it to match your team’s engineering culture
* Extend it with additional rules or learning structures
* Combine it with other documentation systems (e.g. ADRs, architecture docs)

---

## How to Extend It

When creating a derivative or adapted version:

### Keep the core principles intact:

* Problem-first thinking
* Constraints-based reasoning
* Tradeoff-driven decision making
* Code dissection by ownership and behavior

---

### You may customize:

* Depth levels in code dissection
* Teaching tone (strict, casual, mentor-style, etc.)
* Domain-specific extensions (e.g. backend, ML systems, mobile apps)
